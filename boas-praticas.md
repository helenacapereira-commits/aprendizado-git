Boas práticas para usar Git e GitHub
1 - Controle de versão é um sistema que registra alterações em um arquivo ou conjunto de arquivos ao longo do tempo, permitindo que você retorne a versões específicas mais tarde. É importante porque ajuda a gerenciar e acompanhar mudanças no código, facilita a colaboração entre desenvolvedores e protege contra a perda de dados.
2 - Branches são ramificações no repositório que permitem que desenvolvedores trabalhem em diferentes partes de um projeto simultaneamente. Cada branch pode ter seu próprio conjunto de mudanças, e as alterações podem ser mescladas de volta à branch principal quando estiverem prontas. Isso facilita o desenvolvimento paralelo e a experimentação sem afetar o código principal.
3 - Boas práticas para colaboração incluem a criação de branches específicas para cada tarefa, a realização de commits pequenos e específicos, a comunicação eficaz entre membros da equipe, e a revisão de código para garantir a qualidade. Além disso, é importante evitar a inclusão de arquivos sensíveis nos repositórios e seguir convenções de nomenclatura consistentes.
4 - Um merge é o processo de integrar mudanças de uma branch para outra, geralmente da branch de desenvolvimento para a branch principal. No Git, isso é feito usando o comando `git merge`, que combina o histórico de commits de ambas as branches. Conflitos podem ocorrer se houver mudanças conflitantes, e eles devem ser resolvidos manualmente.
